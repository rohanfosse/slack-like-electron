# Task: Frontend DMs etudiant-etudiant

## Description

Ajouter dans la sidebar DM un bouton "+" permettant de rechercher un etudiant par nom dans la meme promo, et adapter `openDm` pour supporter les conversations etudiant-etudiant.

## Acceptance Criteria

- [ ] Un bouton "+" est visible dans la section DM de la sidebar
- [ ] Le clic sur "+" ouvre un champ de recherche filtrant les etudiants de la meme promo
- [ ] La recherche appelle `GET /api/students/search?promoId=X&q=nom`
- [ ] Le clic sur un resultat ouvre la conversation DM avec cet etudiant
- [ ] `openDm` calcule `dm_student_id = min(myId, peerId)` pour les conversations etudiant-etudiant
- [ ] L'UI est coherente avec le design existant des DMs

## Technical Details

- Modifier : `composables/useSidebarDm.ts` — ajouter bouton "+" et logique de recherche
- Modifier : `stores/app.ts` — adapter `openDm` pour calculer `min(myId, peerId)` quand les deux sont etudiants
- L'endpoint de recherche retourne `{ id, firstName, lastName, promoName }`
- La recherche est debounced (300ms) pour eviter les appels excessifs
- Le composant de recherche utilise un dropdown avec les resultats filtres

## Effort Estimate

- Size: L
- Hours: 3

## Definition of Done

- [ ] Code implemente et fonctionnel
- [ ] Le bouton "+" et la recherche fonctionnent correctement
- [ ] La navigation vers une conversation etudiant-etudiant fonctionne
- [ ] L'UI est responsive et accessible
- [ ] Aucune regression sur les DMs existants

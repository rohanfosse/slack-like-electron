# Task: Tests Phase 1 robustesse DMs

Ecrire les tests unitaires couvrant les fonctionnalites de la Phase 1 : queue offline (`dmQueue`), retry avec backoff exponentiel, search limit 200, et validation du destinataire.

## Acceptance Criteria

- [ ] Tests unitaires pour `dmQueue` : save, load, flush, clear
- [ ] Test FIFO : verifier que les messages les plus anciens sont supprimes quand max 50 est atteint
- [ ] Tests retry : mock reseau en echec, verifier 3 tentatives avec delais corrects
- [ ] Test fallback queue : apres 3 echecs, le message est dans la queue
- [ ] Tests search limit : requete avec > 200 messages retourne exactement 200 + `truncated: true`
- [ ] Tests validation : `dmStudentId` inexistant retourne erreur 400
- [ ] Couverture >= 80% sur les fichiers modifies

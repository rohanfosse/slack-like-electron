# Task: Backend DMs etudiant-etudiant

## Description

Adapter le backend pour supporter les DMs entre etudiants de la meme promo. Lever le blocage `dmPeerId >= 0`, ajouter un check de meme promo, utiliser la boite partagee `dm_student_id = min(A, B)`, adapter le middleware `requireDmParticipant`, et adapter `getRecentDmContacts` pour inclure les contacts etudiants.

## Acceptance Criteria

- [ ] Le blocage `dmPeerId >= 0` est leve dans `validateDm`
- [ ] Un etudiant peut envoyer un DM a un camarade de la meme promo
- [ ] Un etudiant ne peut PAS envoyer un DM a un etudiant d'une autre promo (erreur 403)
- [ ] La boite DM est calculee comme `dm_student_id = min(sender, recipient)`
- [ ] `requireDmParticipant` autorise l'acces si l'utilisateur est le box owner OU un peer dans la boite
- [ ] `getRecentDmContacts` inclut les contacts etudiants dans la liste

## Technical Details

- Modifier : `server/services/messages.js` — lever blocage, ajouter check promo, logique min(A,B)
- Modifier : `server/middleware/authorize.js` — adapter `requireDmParticipant` avec `isParticipant(boxId, userId)`
- Modifier : `server/db/models/messages.js` — adapter `getRecentDmContacts` pour les boites etudiants
- Le check promo utilise une jointure `students JOIN promos` pour verifier que les deux etudiants partagent la meme promo
- La boite partagee evite les doublons : A→B et B→A utilisent la meme boite

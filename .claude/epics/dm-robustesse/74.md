# Task: Search limit 200 + validation destinataire

## Description

Ajouter un `LIMIT 200` sur la requete `searchDmMessages` cote serveur avant le dechiffrement, et verifier que le `dmStudentId` existe dans la table `students` avant d'inserer un message DM.

Retourner un flag `truncated: true` si les resultats sont limites a 200. Les messages d'erreur sont en francais.

## Acceptance Criteria

- [ ] `searchDmMessages` applique `LIMIT 200 ORDER BY created_at DESC` avant dechiffrement
- [ ] La reponse inclut un flag `truncated: true` quand les resultats sont coupes
- [ ] `dmStudentId` est verifie dans la table `students` avant insertion
- [ ] Un `dmStudentId` inexistant retourne une erreur 400 avec message francais explicite
- [ ] Les messages d'erreur sont en francais (ex: "Destinataire introuvable")

## Technical Details

- Modifier : `server/db/models/messages.js` — ajouter LIMIT 200 sur `searchDmMessages`
- Modifier : `server/services/messages.js` — verifier existence du destinataire dans `students`
- Le flag `truncated` est ajoute a la reponse de recherche uniquement quand le nombre de resultats atteint 200
- La validation du destinataire utilise une requete simple `SELECT id FROM students WHERE id = ?`

# Task: Tests Phase 2 DMs etudiant-etudiant

## Description

Ecrire les tests d'integration couvrant les fonctionnalites de la Phase 2 : DMs etudiant-etudiant avec verification de promo, boite partagee, middleware d'acces, et chiffrement.

## Acceptance Criteria

- [ ] Test : etudiant envoie DM a un camarade de meme promo — succes 200
- [ ] Test : etudiant envoie DM a un etudiant d'une autre promo — erreur 403
- [ ] Test : la boite partagee `min(A,B)` est identique dans les deux sens (A→B et B→A)
- [ ] Test : `requireDmParticipant` autorise l'acces aux deux participants de la boite
- [ ] Test : `requireDmParticipant` bloque un etudiant non-participant
- [ ] Test : le chiffrement fonctionne correctement pour les DMs etudiant-etudiant
- [ ] Test : `getRecentDmContacts` inclut les contacts etudiants
- [ ] Couverture >= 80% sur les fichiers modifies en Phase 2

## Technical Details

- Tests d'integration pour `server/services/messages.js` : scenarii meme promo / cross promo
- Tests pour `server/middleware/authorize.js` : acces boite partagee
- Tests pour `server/db/models/messages.js` : contacts etudiants dans la liste recente
- Tests frontend pour `composables/useSidebarDm.ts` : recherche, selection, ouverture conversation
- Utiliser des fixtures avec au moins 2 promos et 3 etudiants pour couvrir tous les cas

# Task: Queue offline + retry envoi DM

## Description

Creer `utils/dmQueue.ts` pour gerer une queue de messages DM en mode offline, et ajouter un mecanisme de retry avec backoff exponentiel (1s, 3s, 9s) dans la methode `sendMessage()` du store messages.

La queue utilise `safeStorage` avec la cle `cc_dm_queue`, stocke un maximum de 50 messages en mode FIFO (les plus anciens sont supprimes en premier si la limite est atteinte).

Le flush de la queue se declenche automatiquement sur l'event `online` du navigateur et lors de la reconnexion du socket.

## Acceptance Criteria

- [ ] `utils/dmQueue.ts` exporte les fonctions `save`, `load`, `flush`, `clear`
- [ ] La queue persiste via `safeStorage` avec la cle `cc_dm_queue`
- [ ] Maximum 50 messages en FIFO — les plus anciens sont supprimes quand la limite est depassee
- [ ] `sendMessage()` dans le store messages tente 3 retries avec backoff exponentiel (1s, 3s, 9s)
- [ ] Apres 3 echecs, le message est ajoute a la queue offline
- [ ] La queue est flushee automatiquement sur l'event `online`
- [ ] La queue est flushee automatiquement lors de la reconnexion socket
- [ ] Des toasts informatifs sont affiches a chaque etape (retry, mise en queue, flush)

## Technical Details

- Nouveau fichier : `utils/dmQueue.ts`
- Modifier : `stores/messages.ts` (methode `sendMessage`)
- Ecouter `window.addEventListener('online', ...)` et l'event socket `connect`
- Le backoff est exponentiel : 1s, 3s, 9s (facteur 3)
- Les messages en queue contiennent toutes les infos necessaires pour re-envoyer (destinataire, contenu chiffre, metadata)

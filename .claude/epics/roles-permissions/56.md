# Task: Module permissions centralise (shared)

## Description

Creer un module `shared/permissions.js` utilisable cote serveur ET cote renderer. Ce module est la **source unique de verite** pour la hierarchie de roles et les verifications de permissions.

Contenu :
```js
const ROLE_LEVELS = { student: 0, ta: 1, teacher: 2, admin: 3 }

function hasRole(userRole, requiredRole) {
  return (ROLE_LEVELS[userRole] ?? -1) >= (ROLE_LEVELS[requiredRole] ?? Infinity)
}

function isSystemAdmin(role) { return role === 'admin' }

const ADMIN_MODULES = {
  stats: 'teacher', feedback: 'teacher', moderation: 'teacher', users: 'teacher',
  security: 'admin', maintenance: 'admin', deploy: 'admin',
  sessions: 'admin', settings: 'admin', audit: 'admin',
}

function canAccessModule(userRole, moduleName) {
  const required = ADMIN_MODULES[moduleName]
  return required ? hasRole(userRole, required) : false
}
```

## Acceptance Criteria

- [ ] `shared/permissions.js` exporte `ROLE_LEVELS`, `hasRole`, `isSystemAdmin`, `ADMIN_MODULES`, `canAccessModule`
- [ ] `hasRole('ta', 'teacher')` retourne `false`
- [ ] `hasRole('admin', 'teacher')` retourne `true`
- [ ] `hasRole('teacher', 'teacher')` retourne `true`
- [ ] `canAccessModule('teacher', 'security')` retourne `false`
- [ ] `canAccessModule('admin', 'security')` retourne `true`
- [ ] `canAccessModule('teacher', 'stats')` retourne `true`
- [ ] Le module est importable depuis le serveur (`require`) et le renderer (`import`)
- [ ] Tests unitaires couvrent tous les cas limites

## Technical Details

Placer dans `shared/permissions.js` avec un export CommonJS + ESM compatible (ou double export). Le renderer utilise electron-vite qui supporte les imports depuis des fichiers hors de `src/` via un alias.

Alternative : placer dans `server/` et creer un miroir dans `src/renderer/src/utils/permissions.ts` qui re-exporte les constantes. Plus simple pour electron-vite.

# Task: Schema DB — tables Projet + migration role admin

## Description

Ajouter les nouvelles tables et migrer le schema existant dans `server/db/schema.js` :
1. Etendre le CHECK constraint de `teachers.role` pour inclure `'admin'`
2. Creer la table `projects` (id, promo_id, name, description, channel_id, deadline, created_by, created_at)
3. Creer la table `project_travaux` (project_id, travail_id) — liaison devoirs-projets
4. Creer la table `project_documents` (project_id, document_id) — liaison documents-projets
5. Creer la table `teacher_projects` (teacher_id, project_id, can_grade, assigned_at) — remplace teacher_channels
6. Creer la table `teacher_promos` (teacher_id, promo_id) — liaison enseignant-promo

## Acceptance Criteria

- [ ] `teachers.role` accepte `'admin'`, `'teacher'`, `'ta'`
- [ ] Les 5 nouvelles tables existent avec les bonnes contraintes FK
- [ ] Les index necessaires sont crees (project_travaux, teacher_projects)
- [ ] La migration s'execute sans erreur sur une DB existante
- [ ] Le seed de demo cree au moins 1 admin, 1 teacher, 1 TA, 2 projets

Utiliser le systeme de migration par version existant (v42+ ou le prochain numero libre).

# Task: Refactor middleware + IPC backend (129 occurrences)

## Description

Remplacer les guards hardcodes par le systeme hierarchique dans tout le backend.

### Middleware (`server/middleware/authorize.js`)

1. Importer `hasRole` depuis le module permissions
2. Creer `requireRole(minRole)` — middleware generique qui remplace `requireTeacher`
3. Creer `requireProject(getProjectId)` — verifie que le TA est assigne au projet
4. Creer `requirePromoAdmin(getPromoId)` — verifie que l'enseignant est responsable de la promo
5. Modifier `requireDmParticipant` pour utiliser `teacher_projects` au lieu de `teacher_channels` et **supprimer le fallback permissif** (lignes 90-104)
6. Exporter les nouveaux guards, deprecier les anciens

### IPC (`src/main/ipc/helpers.js`)

1. Creer `handleRole(minRole, channel, fn)` — remplace `handleTeacher`
2. Creer `handleProject(channel, getProjectId, fn)` — scope TA par projet
3. Deprecier `handleTeacher`

### Routes (20 fichiers, 129 occurrences)

Remplacer toutes les occurrences de `requireTeacher` par `requireRole('teacher')` et `handleTeacher` par `handleRole('teacher', ...)` dans :
- `server/routes/` (16 fichiers)
- `src/main/ipc/` (3 fichiers : travaux.js, structure.js, documents.js)

### Admin routes

Modifier `server/routes/admin/index.js` : `requireAdmin` verifie `role === 'admin'` (plus `type === 'teacher'`).

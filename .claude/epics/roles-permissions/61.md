# Task: Panel admin scope — modules systeme vs promo

## Description

Diviser le panel admin en deux tiers : modules systeme (admin global uniquement) et modules promo (enseignant responsable).

### Backend (`server/routes/admin/`)

1. **`index.js`** : remplacer `requireAdmin` actuel par deux middlewares :
   - `requireSystemAdmin` : verifie `role === 'admin'` (modules systeme)
   - `requirePromoAdmin` : verifie `role === 'teacher'` OU `role === 'admin'` ET que l'enseignant est lie a la promo demandee

2. **Routes systeme** (admin global uniquement) :
   - `/admin/security`, `/admin/maintenance`, `/admin/deploy`, `/admin/sessions`, `/admin/settings`, `/admin/audit`

3. **Routes promo** (enseignant de la promo + admin global) :
   - `/admin/stats` — filtrer par promos de l'enseignant
   - `/admin/feedback` — feedback de la promo uniquement
   - `/admin/moderation` — messages des canaux de la promo
   - `/admin/users` — etudiants de la promo uniquement

4. **Filtrage des donnees** : les requetes SQL dans `server/db/models/admin.js` doivent filtrer par promo quand le role est `'teacher'` (pas `'admin'`).

### Frontend (`server/public/admin/`)

Le panel admin est servi en HTML statique. Modifier :
- La navigation pour masquer les modules systeme si le role n'est pas admin
- L'API frontend pour passer le role et filtrer les modules

## Acceptance Criteria

- [ ] Un enseignant peut acceder aux stats de SA promo via le panel admin
- [ ] Un enseignant ne peut PAS acceder aux stats d'une autre promo
- [ ] Un enseignant ne voit PAS les modules systeme dans la navigation
- [ ] Un admin voit TOUS les modules
- [ ] Un TA ne peut acceder a AUCUN module admin (403)
- [ ] Les donnees retournees sont filtrees par promo pour les enseignants
- [ ] L'admin voit les donnees de toutes les promos

## Technical Details

Le panel admin utilise `server/public/admin/` (HTML/JS statique servi par Express). Les appels API passent par `/api/admin/*` avec le token JWT.

L'info du role est disponible dans le JWT (`req.user.type`). Pour les promos de l'enseignant, utiliser `teacher_promos`.

Attention : `getAdminStats()` dans `admin.js` fait des requetes globales. Il faut ajouter un parametre `promoIds` optionnel pour filtrer.

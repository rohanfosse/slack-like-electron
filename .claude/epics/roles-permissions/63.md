# Task: Tests de regression + validation etudiant

## Description

Ecrire et executer les tests de regression pour garantir que la refonte des permissions ne casse rien pour les etudiants et que le nouveau systeme fonctionne correctement.

### Tests unitaires — module permissions

- `hasRole()` : tester toutes les combinaisons de roles (16 cas)
- `canAccessModule()` : tester chaque module pour chaque role
- `isSystemAdmin()` : true pour admin, false pour les autres

### Tests integration — middleware

- `requireRole('teacher')` : accepte admin et teacher, bloque ta et student
- `requireRole('ta')` : accepte admin, teacher, ta, bloque student
- `requireRole('admin')` : accepte admin uniquement
- `requireProject()` : TA avec projet assigne = OK, TA sans = 403
- `requirePromoAdmin()` : enseignant de la promo = OK, autre enseignant = 403
- `requireDmParticipant()` : TA sans assignation = 403 (plus de fallback)

### Tests integration — routes admin

- Admin accede a tous les modules (200)
- Enseignant accede aux modules promo (200) mais pas systeme (403)
- TA accede a aucun module (403)
- Enseignant ne voit que les donnees de sa promo

### Tests integration — modele Projet

- CRUD projets
- Assignation/desassignation TA
- Liaison devoirs et documents
- Migration des donnees existantes

### Tests regression — parcours etudiant

Verifier que les etudiants peuvent toujours :
- [ ] Se connecter
- [ ] Voir leur dashboard
- [ ] Acceder aux canaux de leur promo
- [ ] Voir et soumettre des devoirs
- [ ] Envoyer et recevoir des messages
- [ ] Recevoir des notifications
- [ ] Consulter leurs notes
- [ ] Acceder aux DMs

### Tests regression — parcours TA

- [ ] Se connecter en tant que TA
- [ ] Voir uniquement les projets assignes
- [ ] Corriger un devoir dans un projet assigne
- [ ] Ne PAS voir les projets non-assignes
- [ ] Recevoir une notification d'assignation

## Acceptance Criteria

- [ ] 100% des tests unitaires passent (module permissions)
- [ ] 100% des tests integration passent (middleware, routes, modele)
- [ ] Zero regression sur le parcours etudiant
- [ ] Le TA sans assignation a zero acces (teste explicitement)
- [ ] Le fallback permissif n'existe plus (teste explicitement)
- [ ] Couverture > 80% sur le module permissions et le middleware authorize

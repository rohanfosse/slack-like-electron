# Task: UI Projet — CRUD enseignant + sidebar TA

## Description

Creer l'interface frontend pour la gestion des projets par les enseignants et l'affichage scope pour les TAs.

### Vue enseignant : gestion des projets

1. **Liste des projets** dans la sidebar ou une vue dediee :
   - Afficher les projets de la promo active
   - Bouton "Creer un projet" (enseignant uniquement)
   - Pour chaque projet : nom, deadline, nombre de devoirs, TAs assignes

2. **Modal creation/edition de projet** :
   - Champs : nom, description, deadline
   - Selection des devoirs a inclure dans le projet
   - Selection des documents a inclure
   - Creation automatique d'un canal dedie (optionnel)

3. **Assignation TA** :
   - Dans la fiche projet, section "Intervenants"
   - Liste des TAs disponibles avec bouton "Assigner"
   - TAs assignes avec bouton "Retirer"
   - L'assignation declenche une notification automatique

### Vue TA : sidebar filtree

1. **Sidebar TA** : ne montrer que les canaux/projets assignes
   - Filtrer `taChannelIds` dans le store pour ne montrer que les canaux des projets assignes
   - Grouper par projet dans la sidebar

2. **Dashboard TA** : widget "Mes projets" montrant les projets assignes avec les devoirs a corriger

### Vue etudiant : inchangee

Les etudiants ne voient pas les projets comme entite — ils voient les devoirs et canaux comme avant. Le projet est une organisation interne enseignant/TA.

## Acceptance Criteria

- [ ] L'enseignant peut creer un projet dans sa promo
- [ ] L'enseignant peut modifier/supprimer un projet
- [ ] L'enseignant peut assigner/desassigner des TAs
- [ ] Le TA recoit une notification a l'assignation
- [ ] La sidebar du TA ne montre que les canaux des projets assignes
- [ ] Le dashboard du TA montre ses projets et les devoirs a corriger
- [ ] L'etudiant ne voit aucun changement dans son experience
- [ ] Un TA sans projet voit une sidebar vide (pas d'acces)
- [ ] L'enseignant peut lier des devoirs existants a un projet

## Technical Details

Composants a creer :
- `ProjectListPanel.vue` — liste des projets de la promo
- `ProjectModal.vue` — creation/edition de projet
- `ProjectTaAssignment.vue` — gestion des TAs du projet

Composables :
- `useProjects()` — CRUD projets via `window.api`
- `useTaProjects()` — projets assignes au TA courant

Store : ajouter `taProjectIds` dans `appStore` (charge au login pour les TAs).

La sidebar (`Sidebar.vue`) doit filtrer les canaux en fonction des projets assignes au TA. Le filtre actuel par `taChannelIds` est remplace par un filtre derive des projets.

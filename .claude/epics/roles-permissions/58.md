# Task: Modele Projet — CRUD + assignation TA

## Description

Creer le modele `server/db/models/projects.js` et les routes associees pour gerer les projets et l'assignation des TAs.

### Modele (`server/db/models/projects.js`)

```js
// CRUD Projets
getProjectsByPromo(promoId)
getProjectById(id)
createProject({ promoId, name, description, channelId, deadline, createdBy })
updateProject(id, { name, description, deadline })
deleteProject(id)

// Liaison devoirs/documents
addTravailToProject(projectId, travailId)
removeTravailFromProject(projectId, travailId)
getProjectTravaux(projectId)
addDocumentToProject(projectId, documentId)
getProjectDocuments(projectId)

// Assignation TA
assignTaToProject(teacherId, projectId)
unassignTaFromProject(teacherId, projectId)
getProjectTas(projectId)
getTaProjects(teacherId)  // projets assignes a un TA

// Liaison enseignant-promo
getTeacherPromos(teacherId)
assignTeacherToPromo(teacherId, promoId)
```

### Routes (`server/routes/projects.js`)

- `GET /api/projects/:promoId` — lister les projets d'une promo
- `POST /api/projects` — creer un projet (teacher+)
- `PUT /api/projects/:id` — modifier un projet (teacher+)
- `DELETE /api/projects/:id` — supprimer un projet (teacher+)
- `POST /api/projects/:id/assign-ta` — assigner un TA au projet
- `DELETE /api/projects/:id/unassign-ta/:teacherId` — retirer un TA
- `GET /api/projects/:id/travaux` — devoirs du projet
- `GET /api/projects/:id/documents` — documents du projet

### IPC handlers (`src/main/ipc/projects.js`)

Miroir des routes pour l'acces Electron via `window.api`.

## Acceptance Criteria

- [ ] CRUD projets fonctionnel (creer, lire, modifier, supprimer)
- [ ] Assignation/desassignation de TAs a un projet
- [ ] Notification automatique au TA lors de l'assignation (via le systeme de notif existant)
- [ ] Liaison devoirs <-> projets (ajout/suppression)
- [ ] Liaison documents <-> projets
- [ ] Un TA ne peut pas creer de projet (guard `requireRole('teacher')`)
- [ ] Routes protegees par les bons guards

## Technical Details

Le systeme de notification existant (`appStore.addNotification()` cote frontend, socket events cote serveur) est reutilise pour notifier le TA.

A la creation d'un projet, un canal dedie peut etre cree automatiquement (optionnel, si `channelId` est null).

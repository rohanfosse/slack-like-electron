# Task: Migration donnees — devoirs vers projets + teacher_channels vers teacher_projects

## Description

Migrer toutes les donnees existantes vers le nouveau modele Projet. Cette migration s'execute automatiquement lors du demarrage de l'app (dans le systeme de migration par version de `schema.js`).

### Etape 1 : Migrer les devoirs dans des projets par defaut

Pour chaque devoir (`travaux`) existant qui n'a pas encore de projet :
1. Creer un projet par defaut dans la meme promo, nomme d'apres le devoir
2. Inserer la liaison dans `project_travaux`

```sql
-- Pour chaque travail sans projet :
INSERT INTO projects (promo_id, name, created_by, created_at)
  SELECT t.promo_id, t.title, t.created_by, t.created_at
  FROM travaux t
  WHERE t.id NOT IN (SELECT travail_id FROM project_travaux);

INSERT INTO project_travaux (project_id, travail_id)
  SELECT p.id, t.id
  FROM projects p
  JOIN travaux t ON t.title = p.name AND t.promo_id = p.promo_id
  WHERE t.id NOT IN (SELECT travail_id FROM project_travaux);
```

### Etape 2 : Migrer teacher_channels vers teacher_projects

Pour chaque assignation dans `teacher_channels` :
1. Trouver les projets dont le canal correspond au `channel_id`
2. Creer les entrees correspondantes dans `teacher_projects`

### Etape 3 : Migrer les enseignants vers teacher_promos

Pour chaque enseignant (`role = 'teacher'`) :
1. Deduire ses promos a partir des canaux ou il a du contenu
2. Inserer dans `teacher_promos`

### Etape 4 : Promouvoir le premier enseignant en admin

Si aucun admin n'existe, promouvoir le premier enseignant (id=1) en admin.

### Etape 5 : Drop teacher_channels

Apres verification que toutes les donnees sont migrees :
```sql
DROP TABLE IF EXISTS teacher_channels;
```

## Acceptance Criteria

- [ ] Tous les devoirs existants appartiennent a un projet
- [ ] Aucun devoir orphelin (`SELECT ... NOT IN project_travaux` retourne 0)
- [ ] Les assignations teacher_channels sont converties en teacher_projects
- [ ] Au moins 1 admin existe apres migration
- [ ] Les enseignants sont lies a leurs promos dans teacher_promos
- [ ] La table teacher_channels est supprimee
- [ ] L'app demarre et fonctionne normalement apres migration
- [ ] Les donnees de demo (seed) sont a jour avec le nouveau modele

## Technical Details

La migration est idempotente : si elle est relancee, elle ne cree pas de doublons (verification `NOT IN` ou `INSERT OR IGNORE`).

Mettre a jour le seed de demo (`server/db/seed.js`) pour creer des projets, des liaisons, et des assignations TA.
